# Архив Chimbal

Статический сайт-архив статей с подсветками, комментариями и запросами на
разбор. Полностью статика — своего бэкенда нет, всё держится на относительных
путях и публичных API.

Раньше был монолитом на самописном рантайме (`support.js`). Разбит на отдельные
файлы по той же схеме, что и лендинг `ncrc-x-chimbal`: без билд-степа, React и
Babel грузятся с CDN, JSX транспилится прямо в браузере.

## Структура

```
index.html              точка входа: CDN-вендоры, шрифты, css, скрипты
css/styles.css          глобальные стили + классы .arc-*
assets/ava.webp         аватарка (та же, что на лендинге)
js/annotations.js       window.Anno — палитра, чистка HTML статьи, наложение пометок
js/requests.js          window.ReqCounter — счётчик «запросов на разбор» (Abacus)
js/starfield.jsx        компонент Starfield (canvas-фон)
js/archive.jsx          компонент Archive — само приложение + монтирование
data/
  articles.json         общий индекс всех статей
  reviews.json          карта статусов разбора { "<id>": {reviewed, requestsAtReview} }
  articles/
    <id>_<slug>/
      index.html         обработанный HTML статьи
      meta.json          реальные заголовок и дата
      images/            картинки статьи
      annotations.json   пометки администратора (необязательно)
```

## Как устроено

- **Стек.** React + ReactDOM + Babel-standalone с CDN (версии запинены). Никакой
  сборки: `.jsx` подключаются как `text/babel` и транспилируются в браузере.
  Компоненты и помощники регистрируются глобалами (`window.Starfield`, `Archive`,
  `Anno`, `ReqCounter`) — один-в-один подход лендинга.
- **Данные.** `articles.json` — индекс (`id`, `slug`, `url`, `folder`, `title`,
  `date`, `tags`, `local`). Тело статьи и картинки лежат в
  `data/articles/<id>_<slug>/` и дочитываются по требованию; `Anno.clean` чистит
  исходный HTML под вёрстку архива.
- **Пометки (аналог community notes).** В админ-режиме — подсветки и комментарии
  к выделенному тексту. Это и есть «разбор» статьи; хранятся в `annotations.json`
  рядом со статьёй.
- **Запросы на разбор.** У неразобранной статьи — кнопка «Хочу разбор ✦» и
  счётчик. Динамику считает публичный API
  **[Abacus](https://abacus.jasoncameron.dev)** (ключ на статью, дедуп «один
  человек = один запрос» флагом в `localStorage`). Статус разбора лежит в общей
  карте `data/reviews.json`: в списке у каждой статьи значок ✓/?, рядом с поиском
  — фильтр по статусу.

## Почему так

- **Чистая статика без бэкенда** — простой хостинг, нечего ломаться на сервере;
  вся динамика (счётчики) вынесена в публичный Abacus.
- **Разбит из монолита** — единый `index.html` + `support.js` тяжело
  поддерживать; разбивка по концернам (стили / помощники / фон / приложение)
  читается проще и совпадает со структурой лендинга.
- **Статусы и пометки — JSON в репозитории.** Раз бэкенда нет, «правда» админа
  живёт прямо в данных (`annotations.json`, `reviews.json`), а `localStorage` —
  лишь черновик до того, как изменения попадут в репозиторий.
