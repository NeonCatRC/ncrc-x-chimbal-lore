# Как работают правки (комментарии и разборы)

Публичный `web` (nginx) монтирует контент и overlay **только `:ro`** — он ничего
не пишет. Пишет лишь **admin-api** на `127.0.0.1:8090`, и только в **overlay**.
Контент-архив неизменяем в принципе.

## Где что лежит

| место | что | когда меняется |
|-------|-----|----------------|
| `localStorage` твоего браузера | черновик правок (`chimbal.anno.v1.<id>`, `chimbal.review.v1.<id>`, флаг `chimbal.admin.v1`) | сразу при каждой правке |
| overlay на сервере | опубликованное (`overlay/annotations/<id>.json`, `overlay/reviews.json`) | по кнопке «Сохранить» (через admin-api) |
| браузер читателя | ничего не пишет, только читает overlay | — |

## Пример: добавить комментарий к статье 257059

0. На сервере поднят admin-api: `docker compose --profile admin up -d admin-api`.
   С ноута туннель: `ssh -L 8080:localhost:8080 -L 8090:localhost:8090 user@server`.
   Открыть **`http://localhost:8080`** (через туннель, не публичный домен).
1. Выбрать статью 257059 → «Режим редактора» (ставит `chimbal.admin.v1=1`).
2. Выделить фразу → тулбар → цвет + «коммент» → текст → Сохранить заметку.
3. **Сейчас:** правка только в `localStorage` твоего браузера. Никто не видит.
   В панели горит «● не сохранено».
4. **«💾 Сохранить»** → `POST http://localhost:8090/annotations/257059` →
   admin-api пишет `overlay/annotations/257059.json`. Индикатор гаснет.
5. Читатель открывает статью → nginx отдаёт обновлённый файл → видит правку.
   Перезапуск не нужен.

## Фолбэк и нюансы

- **admin-api недоступен / нет туннеля / открыт публичный https** → «Сохранить»
  не достучится → авто-фолбэк **«↓ Экспорт»** (скачать `annotations.json`),
  кладёшь вручную в `OVERLAY_DIR/annotations/257059.json` (scp/rsync).
- **«↑ Импорт»** — перенос черновика между браузерами (грузит в localStorage),
  это не публикация.
- **«Сбросить»** — выкинуть локальный черновик, вернуться к версии из overlay.
- **Готча:** если у читателя в localStorage уже есть свои правки этой статьи —
  он видит их, а не опубликованный файл, пока не нажмёт «Сбросить». Для читателя,
  который ничего не правил, видна опубликованная версия.
- **Разборы** — так же, но один общий файл `overlay/reviews.json`, кнопка
  «💾 Сохранить разборы».

## Безопасность

- `web`: контент `:ro` + overlay `:ro` — сервер физически не пишет.
- `admin-api`: монтирует **только** overlay `:rw` (контент недоступен), слушает
  `127.0.0.1`, валидирует id (только цифры, без path traversal), опц. `ADMIN_TOKEN`.
- Любой посетитель может включить «режим редактора» у себя — но изменит лишь свой
  localStorage; публикация требует доступа к overlay (admin-api на localhost).
