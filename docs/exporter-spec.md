# Спека контейнера-экспортёра (для агента-реализатора)

Задача: периодически тянуть новые статьи блога **overclockers.ru/blog/Hard-Workshop**
и складывать в том контента в формате, который уже понимает приложение. Скрапинг —
ваша часть; ниже жёсткий контракт на вход/выход и взаимодействие с `web`.

## Взаимодействие двух контейнеров

```
exporter  --(:rw)-->  [ том CONTENT_DIR ]  <--(:ro)--  web (nginx)
```

- Общий ресурс — **только том контента** (`CONTENT_DIR`). Прямой сети между
  контейнерами нет; координация через файловую систему.
- exporter монтирует контент **`/data:rw`**. **НЕ** монтирует overlay и образ app.
- web монтирует тот же том **`:ro`** и читает «вживую» (перезапуск не нужен).

### Инварианты (обязательно)

1. **Атомарность:** каждый файл писать во временный + `os.replace`. Сначала
   полностью записать папку статьи (images → index.html → meta.json), и **только
   потом** выставить запись в `articles.json` (`local: true`). Иначе web покажет
   статью без контента.
2. **Права 0644** на все файлы (nginx — другой uid; 0600 даст 403).
3. **Идемпотентность:** пропускать id, у которых уже `local: true` и есть папка.
   Ничего не удалять и не перезаписывать в существующих статьях.
4. **Только контент:** не трогать `overlay/`, не писать вне `/data`.
5. **Вежливость к источнику:** задержка между запросами, User-Agent, ретраи.

## Выходной контракт

### Дерево
```
/data/
├── articles.json                      # общий список (обновляется)
└── articles/<id>_<slug>/
    ├── index.html                     # тело статьи
    ├── meta.json
    └── images/<name>                  # сжатые картинки
```
`folder = "<id>_<slug>"`, slug — транслит заголовка (как в существующих папках).

### articles.json (массив; добавить/обновить запись статьи)
```json
{
  "id": "257059",
  "slug": "prodolzhaju-razbor-...",
  "url": "https://overclockers.ru/blog/Hard-Workshop/show/257059/prodolzhaju-...",
  "folder": "257059_prodolzhaju-razbor-...",
  "title": "Продолжаю разбор ...",
  "date": "2026-06-06T18:41:00+03:00",
  "tags": ["Installer-SH", "Ремонт / DIY", "ИИ / Нейросети"],
  "local": true
}
```
`date` — ISO8601 c таймзоной или `null`. `local: true` = контент выложен.

### meta.json
```json
{ "id": "257059", "title": "...", "date": "2026-06-06T18:41:00+03:00",
  "url": "https://...", "images": 33 }
```

### index.html
- Тело статьи; картинки — **относительными** путями `<img src="images/<name>">`
  (приложение само префиксует `data/articles/<folder>/` и вешает lazy/стили).
- Приложение в рантайме вырезает `script, style, h4, #video-container,
  [id^="yandex_rtb"], .pip-mat-inner, .ad-sticly__parent`, первый `h1`, `.meta`
  и комментарии. Чистить рекламу самим — желательно (меньше размер), но не строго.
- Ссылки overclockers вида `<a data-link="..." data-desc="...">` оставлять как есть
  (приложение проставит href из `data-link`).

### Картинки (как в архиве — единый вид)
- Даунскейл по длинной стороне до **`MAX_IMAGE_SIDE` (1000)**, не апскейлить.
- JPEG: перекодировать **q=`JPEG_QUALITY` (72)**, progressive, optimize, без EXIF.
- PNG: адаптивно — если ≤65536 цветов, квантовать до 256 (палитра); иначе
  truecolor + optimize.
- Имя и расширение сохранять. Записывать результат, только если он **меньше**
  оригинала (иначе оставить исходный).
- `page_raw.html` (сырой HTML) — опционально; web его не отдаёт (запрещён), git игнорит.

## Параметры (env)

| env | по умолч. | назначение |
|-----|-----------|------------|
| `CONTENT_DIR` (монтаж → `/data`) | — | том контента (:rw) |
| `BLOG_URL` | `https://overclockers.ru/blog/Hard-Workshop` | источник списка |
| `UPDATE_INTERVAL` | `21600` | период опроса, сек |
| `MAX_IMAGE_SIDE` | `1000` | даунскейл длинной стороны, px |
| `JPEG_QUALITY` | `72` | качество JPEG |
| `USER_AGENT` | (свой) | заголовок запросов |
| `REQUEST_DELAY_MS` | `1000` | пауза между запросами |
| `MAX_PER_RUN` | `0` (=без лимита) | макс. статей за прогон |
| `DRY_RUN` | `0` | прогон без записи |

## Поток работы (рекомендуемый)

1. Получить список статей блога → id + url + title + tags + date.
2. Отфильтровать новые (нет `local:true` / нет папки).
3. Для каждой (с учётом `MAX_PER_RUN`, `REQUEST_DELAY_MS`):
   скачать страницу → извлечь тело + список картинок → скачать и сжать картинки →
   записать `index.html`, `meta.json` (атомарно, 0644) → обновить запись в
   `articles.json` (`local:true`, атомарно).
4. Спать `UPDATE_INTERVAL`, повторить.

## Образ

`python:3.12-alpine` + Pillow (ресайз) + парсер HTML (например, `selectolax`/`lxml`).
Запуск как сервис с профилем `updater` (шаблон в `deploy/docker-compose.yml`):
```
docker compose --profile updater up -d --build updater
```
